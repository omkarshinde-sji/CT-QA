# Getting Started with CollabAi

Choose your deployment path based on your needs and technical background.

---

## 🟣 Option A: Lovable Remix

**Best for:** Non-technical users, rapid deployment, no infrastructure management

| Pros | Cons |
|------|------|
| ✅ 10-minute setup | ❌ Requires Lovable account |
| ✅ No CLI or terminal needed | ❌ Lovable Cloud pricing applies |
| ✅ Lovable AI included (no API key) | |
| ✅ One-click publish | |
| ✅ Automatic edge function deployment | |
| ✅ Built-in database management UI | |

**[→ Lovable Quickstart Guide](./lovable-quickstart.md)**

---

## 🔵 Option B: Self-Host (GitHub + Supabase)

**Best for:** Developers, enterprise deployments, full control

| Pros | Cons |
|------|------|
| ✅ Full code ownership | ❌ Requires technical knowledge |
| ✅ Use your own AI API keys | ❌ Manual infrastructure setup |
| ✅ Custom hosting (Vercel, AWS, Netlify) | ❌ More maintenance required |
| ✅ No Lovable dependency | ❌ CLI tools needed |
| ✅ Self-managed costs | |

**[→ Self-Host Quickstart Guide](./self-host-quickstart.md)**

---

## ❓ Which Path Should I Choose?

| If you... | Choose |
|-----------|--------|
| Want the fastest setup | Lovable Remix |
| Don't want to use terminal/CLI | Lovable Remix |
| Need to control hosting costs | Self-Host |
| Want to use specific AI models (GPT-4, Claude) | Self-Host |
| Are deploying for an enterprise | Self-Host |
| Want to contribute to the project | Self-Host |
